git log
	Вся история коммитов в репе. Хеш коммита - уникальный идентификатор коммита.
git diff
	Показывает разницу между текущим неотслеживаемым состоянием репы и последним снимком репы.
git diff --staged
	Показывает разницу между текущим отслеживаемым состояниемрепы и последним снимком.
git diff COMMIT_ID
	Показывает разницу между текущим состоянием репы и указанным снимком.

			Указатель HEAD
     comm1     comm2    comm3	
master *---------*------*
			|
			HEAD
Обычно показывает на текущий (последний) коммит. Его можно смещать.

git reset
	имеет 3 режима
		--soft
		--mixed
		--hard


git reset [режим] [желаемый коммит]
git reset --hard
	Возвращает проект к указанному коммиту, при этом полностью удаляет все коммиты после указанного. Самая сильная вариация. Удаляет коммиты безвозвратно
git reset --mixed
	Возвращает проект к указанному коммиту, при этом переводит все коммиты после указанного в unstaged зону.
git reset --soft
	Возвращает проект к указанному коммиту, при этом переводит все коммиты после указанного в staged (зелёную) зону.
git reset по умолчанию вызывается для --mixed и HEAD

git checkout
	Используется для перемещения между коммитами, версиями отдельных файлов, ветками.
Перемещение между коммитами
git checkout [хеш]
git checkout HEAD^^
git checkout HEAD~2
     comm1     comm2    comm3 comm4
master *---------*------*------*
		 |
		HEAD
Состояние проекта полностью перенесено на снимок comm2. При этом ничего не удалилось. В любой момент можем вернуться. Указатель HEAD находится в состоянии DEATACHED. Он отделен от актуальной версии проекта. Любые изменения или коммиты сделанные в этом состоянии удаляются сборщиком мусора при переходе к другому коммиту.
git checkout branch
	Переход к актуальной версии коммита
git checkout [хеш] -- [путь_к_файлу]
	Возвращает файл к версии которая была в указанном коммите.
git checkout -- [путь_к_файлу]
	Возвращает файл к версии, которая была у него в последнем коммите. Работает только для untracked и modified.
git checkout -- .
	Возвращает все файлы в репе к версии в последнем коммите.

Если хочешь удалить отслеживаемые изменения (git add):
	1. git reset - переводим отслеживаемые в untracked
	2. git checkout -- . - удаляем все untracked изменения
